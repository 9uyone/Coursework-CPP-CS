2.	Перевантажив варіант сетерів для Vertex, що приймають об'єкт типу Vertex за універсальним посиланням (lvalue & rvalue).
Правила згортки посилань (reference collapsing):
	T& &	-> T&
	T& &&	-> T&
	T&& &	-> T&
	T&& &&	-> T&&
Одним словом, lvalue reference виграє, якщо присутнє.

3.	Для класу Vertex додав оператор введення >>

4.	Для класу Vertex додав оператор виводу << з універсальним посиланням.
Використовуючи requires встановив обмеження (C++20), щоб не було конфілктів з cout.operator<<
Для старших стандартів (C++11+) можна використати правило SFINAE: "Substitution failure is not an error".
